Data Types, Objects and Literals

In JavaScript every data type is represented as an object with different properties.

But you can use literals and primitive data types to represent values.
These are fixed values, not variables - that you literally provide in your script.

The different data types are:

  • Array
  • Boolean
  • Floating-Point

  • MISSING:

    Integers

  • MISSING:

    Object

  • MISSING:

    String

Array Literals

An array literal exists of at least zero (or more) expressions that represent the array elements. They are enclosed in square brackets ([]).
When you create an array using an array literal, it is initialized with the specified values as its elements and its length is set to the number of arguments specified. 

var name = [ 'Christoph', 'Martens' ];

alert(name[0]); // "Christoph"
alert(name[1]); // "Martens"
alert(name.length); // 2

Initializing Arrays with undefined elements

You can add extra commas between defined elements to add an extra element to the array, which has an undefined value. 

var name = [ 'Christoph', , 'Martens' ]; // no mistake. see below.

alert(name[0]); // "Christoph"
alert(name[1]); // undefined
alert(name[2]); // "Martens"
alert(name.length); // 3

Trailing commas

Trailing commas at the end of the list of elements are ignored, see the below examples for details. 

var name = [ 'Christoph', , 'Martens', ];

... will have no name[3] set.

Multiple trailing commas

Attention

Here's a different behaviour in WebKit and Gecko
var name = [ 'Christoph', , 'Martens', , ];

WebKit behaviour
The array will have no name[3] and no name[4], all trailing commas are ignored.

Gecko behaviour
The array will have name[3] as undefined and no name[4].

Boolean Object

The Boolean Object is a wrapper around the primitive Boolean data type.
It accepts the values true and false

Keep in mind that this wrapper will create an object and not a value. 

var result = new Boolean(value);
alert(typeof result); // "object"

Boolean Literals

The boolean type has two literal values: true and false 

var result = true;
alert(typeof result); // "boolean"

Important

Do not confuse the primitive Boolean (literal) values with the values of the Boolean Object!

The following example will show the difference between the primitive Boolean values and the Boolean Object values. 

var result1 = new Boolean(false); // Boolean object value
if(result1){
	// this part will be executed!
	alert(typeof result1);
	alert(result1);
}

var result2 = false; // primitive Boolean literal
if(result2){
	// this part will NOT be executed!
	alert(typeof result2);
	alert(result2);
}
Explanation:

Any value that is not undefined, null, 0, NaN, or the empty string (""), and any object, including a Boolean object whose value is false, evaluates to true when passed to a conditional statement.

Floating-Point Literals

A floating-point literal can match the following parts:

  • decimal integer which can be signed (+ / -)
  • decimal point
  • fraction (another decimal integer)
  • exponent

The exponent part is an e or E followed by an integer, which can also be signed. A floating-point literal must have at least one digit and either a decimal point or e (well, or E). Here are some floating-point literals examples: 

var eg1 =  3.123;
var eg2 = -3.321;
var eg3 = -3.1E9;
var eg4 =   .1e2;
var eg5 = 2E-12;

String Literals

String literals are enclosed in double (") or single (') quotation marks. A string literal itself can also have zero characters. A string must be delimited by the quotation mark of the same type. That means you will have to start and end with double quotation marks or you will have to start and end with single quotation marks. 

var eg1 = "Foo";
var eg2 = 'Bar';
var eg3 = "1234";
var eg4 = "First line \n Second line";
var eg5 = "Foo's Bar";

As you already know, the String literal is a representation of the native String object. The String object also offers a property called length. 

alert("This is a string".length); // 16
